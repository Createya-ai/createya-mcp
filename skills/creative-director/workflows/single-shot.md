# Workflow: Single shot — один кадр без серии

**Триггер:** «один кадр», «фото для X», «снимок», «постер», «obложка», «hero image», «один image» (без слов про серию/lookbook)

**Главное отличие от других workflow:** **NO hero-loop**. Один кадр — одна генерация (с возможностью итераций). Никаких approval → variations.

## Когда использовать

- Юзер прямо сказал «один»
- Задача очевидно single-shot (постер / cover / OG image / banner)
- Тест концепта / quick exploration

## Цепочка

### Шаг 1 — Бриф

Минимум вопросов:
1. **Если есть референс** → reverse-describe + 1 точечный вопрос
2. **Если нет референса** → 1–2 вопроса максимум (модель если не очевидна, главный визуальный stake)

### Шаг 2 — Модель и промт

`mcp__createya__list_models` → отбери 3 кандидата под задачу → `AskUserQuestion` (если модель не очевидна из брифа).

`mcp__createya__get_model_guide(slug)` → используй `prompt_skeleton` для composition.

### Шаг 3 — Генерация

`run_model` → download через `bash ./scripts/download.sh <url>` → `Read` → vision QA.

### Шаг 4 — Итерации (если просит)

Если юзер просит правки — короткие команды → точечная re-generation:

| Команда юзера | Что делать |
|---------------|------------|
| «теплее» | Добавить «warmer color temperature, golden tint» к промту |
| «холоднее» | Добавить «cooler color temperature, blue undertone» |
| «шире» | Сменить framing на «wide cinematic», убрать subject из центра |
| «ближе» | Сменить framing на «close-up» / «tight crop» |
| «другой ракурс» | Сменить угол: «from below / from above / 3/4 angle» |
| «убери X» | Edit-режим: «remove [X], fill background naturally» |
| «добавь Y» | Edit-режим: «add [Y] to [position]» |
| «больше воздуха» | Добавить negative space, отдалить subject |
| «темнее / атмосфернее» | Снизить exposure, добавить «moody, low-key, deep shadows» |
| «ярче / чище» | Поднять exposure, «high-key, clean, crisp» |

После каждой итерации — снова download + Read + QA.

**Лимит:** 3 итерации, потом стоп — обсуди с юзером направление, не жги кредиты.

## Что НЕ делать

- ❌ НЕ устраивать опросник из 7 блоков для одного кадра
- ❌ НЕ предлагать «давай я сделаю серию» если юзер сказал «один»
- ❌ НЕ генерить 4 варианта одновременно «чтобы выбрать» — это сжигает кредиты, лучше одна сильная итерация чем 4 слабых

## Кредиты

- 1 кадр: 10–25 cr (зависит от модели)
- С итерациями: 30–75 cr (если юзер просит правки)

## Когда переключиться на series

Если по ходу одного кадра юзер говорит «а давай ещё в другом ракурсе / с другим фоном / на другой модели» — это уже **series**. Переключайся на `lookbook.md` или `editorial-series.md`:
1. Текущий кадр становится эталоном
2. Дальше — вариации с `image_urls=[<approved>]`
3. Сообщи юзеру: «Делаю как серию — лочу субъект, меняю [что меняется]»
