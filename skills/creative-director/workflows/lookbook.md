# Workflow: Lookbook — модель + N образов

**Триггер:** «lookbook», «модель в N образах», «фотосессия одежды на модели», «образ + образ», «series outfits»

**Continuity anchors (что lock'ается):**
- Лицо модели (всегда — если юзер дал фото или мы создали character)
- Тело / фигура / возраст
- Light direction (если выбран один — держим во всей серии)
- Цветовая палитра локации (либо одна локация, либо одна палитра)

**Что меняется между кадрами:**
- Одежда (главное)
- Поза
- Кадрирование (close / medium / full body)
- Микро-локация (угол улицы, кусок интерьера) если в одной локации

## Цепочка генераций

### Шаг 1 — Эталон (модель + первый образ)

**Модель:** Nano Banana 2 (или NB Pro Edit если есть фото модели)
**Промт-структура** (из `get_model_guide`):
```
[Описание модели: возраст / черты / hair / тип фигуры]
+ [Outfit #1: материал / цвет / силуэт / детали]
+ [Локация: одна фраза, освещение]
+ [Камера: lens / framing / mood]
+ [Стиль: editorial / commercial / casual]
```

Если есть фото модели — **обязательно** `image_urls=[<face_cdn>]` + в промте «keep the face exactly as in the reference».

**Vision QA после генерации:**
- Лицо узнаваемо (если есть референс)
- Одежда читается (текстура / цвет / логотипы если есть)
- Анатомия рук
- Свет: одно направление, не противоречивый

### Шаг 2 — Approval юзера

Открыть результат через `bash open creative/sessions/<slug>/results/etalon.jpg` → дождаться ответа.

**Без approval не двигаемся.** Если юзер хочет правки — итерируем эталон, не идём к outfits.

### Шаг 3 — Вариации (outfits 2..N)

Для каждого следующего образа:
- `image_urls = [<approved_etalon_cdn>]` — лицо/тело берётся отсюда
- В промт: **«Same model, same face, same body. Change outfit to: [новый образ]. [Что ещё меняем — поза / угол]. Keep: [light / palette]».**
- Skeleton остаётся тем же что в эталоне

**Vision QA каждой вариации:**
- Лицо НЕ изменилось vs эталон (если изменилось — retry с усиленным «keep face exact»)
- Новый outfit реально отличается от предыдущего
- Цвет освещения совпадает с эталоном

## Что спрашивать у юзера

**Обязательно:**
1. Кол-во образов в серии (3 / 5 / 8 / своё)
2. Описание каждого образа — одной строкой («1) jeans + white tee, 2) суит, 3) спортсмен в худи»)
3. Локация(и) — одна или меняется

**Опционально (если не сказал):**
- Стиль: street / editorial / studio / commercial
- Свет: дневной / golden hour / studio softbox

**НЕ спрашивай:**
- Поза каждого кадра — решаешь сам
- Камера каждого кадра — решаешь сам
- Aspect ratio — берёшь из `parameters_schema` модели или MASTER_CONTEXT

## Что НЕ меняется (запиши в session.md)

```
LOCKED:
- Face: <описание или ref ID>
- Body type: <описание>
- Lighting direction: <left/right/top/...>
- Color palette: <warm-natural / cool-clinical / luxury-muted>
```

Перед каждой генерацией — упоминай LOCKED в промте дословно.

## Кредиты

- Эталон: 1× стоимость модели (NB 2 = ~10, NB Pro = ~18)
- Вариации: 1× стоимость на каждую
- Серия из 5 на NB 2 = ~50 cr, на NB Pro = ~90 cr

Покажи юзеру **общую стоимость** ДО старта серии после approval эталона.

## Видео-расширение (когда появятся video модели)

Каждый approved kadr можно будет анимировать через image-to-video (Kling / Veo / Seedance). Запиши в `session.md`:
```
TODO video: shot-01.jpg → image-to-video (когда подключим)
```
