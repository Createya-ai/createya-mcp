# Triage — классификация запроса в первые 3 секунды

**Цель:** до любых вопросов решить — какой это режим работы. От этого зависит всё дальше: грузить ли referencefirst, какой workflow запускать, сколько вопросов задавать.

## 7 режимов

| Режим | Сигналы в запросе | Что делать |
|-------|-------------------|------------|
| **`still`** | «фото», «кадр», «снимок», «один кадр», «postcard» | Прямая генерация. Минимум вопросов (1–2). Без hero-loop если кадр один. |
| **`edit`** | «измени», «убери», «добавь», «замени», «помести в», «поменяй» + есть референс | `image-to-image` модель. Описать референс → точечная правка. Никаких блоков опросника. |
| **`series`** | «серия», «несколько кадров», «lookbook», «N образов», «фотосессия» (без уточнения = несколько) | **Hero-shot loop обязателен.** Эталон → approval → N вариаций с залоченным субъектом. |
| **`character`** | «AI-модель», «персонаж», «character sheet», «9 ракурсов» | Workflow `character-sheet`. Несколько ракурсов одного лица. |
| **`pipeline`** | «товарка для маркетплейса», «UGC отзыв», «полный рекламный креатив» | Многошаговый workflow. Hero → cutout → composite, или модель → outfits, или selfie + product. |
| **`video`** *(пока без моделей)* | «видео», «клип», «reels», «короткое видео», «анимация» | Two-step: сначала still hero (image модель) → ждать когда добавим video модели. Сообщить юзеру про текущее ограничение. |
| **`utility`** *(пока без моделей)* | «вырезать фон», «удалить bg», «прозрачный png» | Пока нет встроенного — предложить сделать через `edit` режим nano-banana с инструкцией «remove background». |

## Алгоритм классификации

```
1. Есть ли референс-изображение в брифе или на диске?
   ├── ДА → почти наверняка `edit` или `series` (hero loop)
   └── НЕТ → один из `still / series / character / pipeline / video`

2. Сколько кадров?
   ├── 1 кадр явно → `still`
   ├── ≥2 кадров явно → `series`
   ├── «AI-персонаж» / «character sheet» → `character`
   └── не сказано → спросить ОДИН раз

3. Видео?
   └── Сейчас сообщи: «Видеомодели подключаем — пока могу сделать starting frame, дальше прицелимся когда добавим Kling / Veo / Seedance»
```

## Примеры триажа

| Запрос юзера | Режим | Почему |
|--------------|-------|--------|
| «Сними фото жёлтого худи на белом фоне» | `still` | один кадр, без серии |
| «Сделай lookbook этого худи на 5 образов» | `series` (workflow `lookbook`) | серия + локнутый товар |
| «Помести этого мужика в офис» + фото мужика | `edit` | прямая правка референса |
| «Создай AI-модель: блондинка 28 лет, 9 ракурсов» | `character` (workflow `character-sheet`) | 9 ракурсов одного лица |
| «Карточка товара для WB: 1 hero + 4 ракурса» | `pipeline` (workflow `product-cutout-pipeline`) | многошаговый ecom |
| «Сделай UGC-отзыв с этим средством» | `pipeline` (workflow `ugc-ad`) | selfie + product |
| «Анимируй этот кадр» | `video` | пока остановись на сообщении про ограничение |
| «Вырежи фон» | `utility` → `edit` через nano-banana | временное обходное решение |

## Правила режима

1. **Решай молча** — не объявляй юзеру «классифицировал как series». Просто действуй по правилам этого режима.
2. **Один режим = один workflow.** Если режим `series` — открыл `workflows/<recipe>.md` подходящий, действуй по нему.
3. **Edit без референса невозможен.** Если юзер сказал «измени фото» а референса нет — сначала спроси «куда положить?» через стандартный intake.
4. **Series без anchor невозможен.** Если режим `series` но не понятно что такое субъект — спроси про субъект, остальное закрой сам.
