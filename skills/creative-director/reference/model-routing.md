# Model routing — task → актуальная модель

**Главное правило:** этот файл — стартовая точка. Перед каждой сессией обязательно `mcp__createya__list_models` чтобы увидеть **актуальный** каталог. Версии моделей меняются — не предлагай юзеру вышедшую из жизни Pro когда уже есть 2.

## Правило актуальности версий

Когда модель называется с цифрой/«Pro»/«Lite»/«2» — **старшая цифра новее**. Примеры (на 2026-05-04, проверяй через `list_models`):

- **Nano Banana 2** новее чем Nano Banana Pro
- **Flux 2** новее чем Flux Pro / Flux 1.1
- **Kling V3 / O3** новее чем V2 / V1
- **Veo 3.1** новее чем Veo 2
- **Seedream 5** новее чем Seedance 4.5

Если в `list_models` есть и старая и новая — **предлагай юзеру только новую**, кроме случая когда у старой явно дешевле и юзер просил бюджет.

## Routing table — task → топ-3 модели

Юзер выбирает САМ из 3 предложенных. Скилл фильтрует список под задачу, не показывает все 148 endpoint'ов сразу.

### Image generation (text-to-image)

| Задача | Топ-3 актуальных моделей | Когда какую |
|--------|-------------------------|-------------|
| Фотореализм / hero shot | Nano Banana 2, Nano Banana Pro, Flux 2 | NB2 — основной выбор; NB Pro — luxury; Flux 2 — детализация одежды |
| С точным текстом в кадре | GPT Image 2, Wan 2.7 Pro, Qwen Image 2 Pro | GPT Image 2 — общий, Wan/Qwen — постеры |
| Художественный стиль / иллюстрация | Midjourney, Recraft V4 Pro, Higgsfield Soul | MJ — киношность, Recraft — графика, Higgsfield — surreal |
| Аниме / мульт | Midjourney (с styles), Higgsfield, Grok Imagine | По брифу — стиль зависит от reference |
| Бюджетная итерация | Nano Banana 2 (дешёвая), Flux Schnell | Когда нужно много вариантов на пробу |

### Image editing (image-to-image, с референсом)

| Задача | Топ-3 моделей | Когда какую |
|--------|---------------|-------------|
| Точечная правка с сохранением субъекта | Nano Banana Pro Edit, GPT Image 2 Edit, Flux Kontext | NB Pro Edit — лучший на людях, GPT — на тексте, Flux Kontext — структурные правки |
| Помещение субъекта в новую сцену | Nano Banana Pro Edit, GPT Image 2 Edit | NB Pro Edit — основной |
| Inpaint / mask-aware | Flux Kontext, Bria Fibo Edit | По наличию маски |
| Удаление объектов | Nano Banana Pro Edit, GPT Image 2 Edit | Используй с anti-pattern: «не описывай что удалить — описывай как должно выглядеть БЕЗ» |

## Стратегия отбора 3-х моделей

**Не просто фильтруй** — формируй три кнопки с разной логикой:

```
Кнопка 1 — РЕКОМЕНДУЮ (твой выбор как CD под конкретную задачу)
Кнопка 2 — БЮДЖЕТ (дешевле, для итераций)
Кнопка 3 — ПРЕМИУМ (если важно качество / финал)
```

Пример для «фотосессия товара luxury»:

```
AskUserQuestion({
  questions: [{
    question: "Какую модель используем?",
    header: "Модель",
    multiSelect: false,
    options: [
      { label: "Nano Banana Pro", description: "Рекомендую — luxury hero shot, ~18 кр/кадр" },
      { label: "Nano Banana 2", description: "Бюджет — те же возможности, ~10 кр/кадр" },
      { label: "Flux 2", description: "Премиум — максимальная детализация ткани, ~25 кр/кадр" }
    ]
  }]
})
```

## Что не делать

- ❌ **Не предлагай 5+ моделей** — три кнопки максимум, иначе юзер залипает
- ❌ **Не предлагай устаревшие версии** (NB Pro если есть NB 2 = новее) — проверяй через `list_models`
- ❌ **Не молчи о цене** — каждая кнопка должна показать `~N кр/кадр` (из `list_models.credits_per_request`)
- ❌ **Не предлагай video-модели до того как они подключены** — сообщай статус честно
